<product_requirements>
The product aims to be a dental practice management software for Madagascar, inspired by Agatha 7, with a strong focus on localization (Malagasy Ariary - MGA, NIF/STAT fields). The initial MVP included Patient Management with basic dental charting, MGA invoicing, and mocked SMS/Mobile Money integrations. The current Node.js/PostgreSQL MVP expanded this to include roles (ADMIN, DENTIST, ASSISTANT, ACCOUNTANT), comprehensive backend entities (Users, Patients, Appointments, Treatments, Procedures, Invoices, Payments, SMS/Audit Logs), and a dashboard with KPIs.

Subsequent expansions included:
1.  **Inventory/Stock Management**: Products (unique SKU, pricing, min_qty, current_qty ≥ 0), stock movements (IN, OUT, ADJUST, qty > 0, prevent negative stock), low-stock alerts, supplier integration (name + NIF/STAT).
2.  **Dental Lab Management**: Lab orders (status flow: CREATED → SENT → IN_PROGRESS → DELIVERED, due_date, total_mga), items, deliveries (delivered_at, delivery_ref), printable PDF for orders (clinic NIF/STAT, MGA).
3.  **Patient Mailing**: Campaigns (audience filters, mock sender, analytics, DRAFT → SCHEDULED → SENT status), compliance (valid email, opt-out flag, rate limiting).
4.  **UI/UX Upgrade**: Modern, medical-themed design (blue/green/white palette), animated UI (sidebar, topbar, dashboard KPIs, charts), dark mode, responsive layout, branding (logo upload).
5.  **Media Management**: Patient profile photos, clinical documents (X-rays), Staff directory (photo, role, contact). APIs for upload/list/delete (mock storage).
6.  **Commercial SaaS Features (Current focus)**: Multi-tenancy (clinic_id for all entities), Subscription system (Essential, Pro, Group plans in MGA, trial mode), Billing (subscription invoices, mock payments), Licensing guard, Super-admin portal, dedicated UI pages.
</product_requirements>
